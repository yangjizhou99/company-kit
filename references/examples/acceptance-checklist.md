# Acceptance Checklist

Use this when a task is in Review.

## Review steps
- Verify outputs against acceptance criteria.
- Check dependencies that may be unblocked if this passes.
- Decide: Pass or Fail.

## If pass
- Set status to Done.
- Add a log entry with the review summary.
- Announce newly Ready tasks.
- Update dependent tasks to Ready if unblocked.

## If fail
- Set status to Rework.
- Record the reason and required changes.
- Reassign or return to the same owner.
