# Task Status Update Checklist

Use this to keep the board current so new members can claim tasks.

## After decomposition
- All tasks created with difficulty and dependencies.
- Status set to Ready for tasks with no unmet dependencies.
- Announce the Ready task list.

## When a task is claimed
- Announce claim.
- Status set to In Progress.
- Log entry added.

## When a task is delivered
- Status set to Review.
- Log entry added with output summary.

## After review
- If pass: status set to Done.
- If fail: status set to Rework with reason.

## When dependencies change
- If a dependency is Done, re-check blocked tasks.
- Move newly unblocked tasks to Ready.
- Announce the updated Ready list.
