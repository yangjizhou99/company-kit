# 用户操作指南

你（用户）在一人公司框架中的角色是**老板 + 调度员**。  
你不写代码、不做分解，只做三件事：**指派、推进、验收**。

---

## 你需要准备什么

| 项目 | 说明 |
|------|------|
| AI 会话 A | 充当 Manager（负责人） |
| AI 会话 B+ | 充当 Worker（员工），可以多个 |
| company-kit/ | 已存在于工作区 |

> **简化模式**：只用一个 AI 会话同时扮演 Manager + Worker，见下方说明。

---

## 操作流程

### 1. 指派负责人（做一次）
在 AI 会话 A 中发送：
```
读取 company-kit/SKILL.md，你是本项目的 Manager。
项目目标：开发一个浏览器贪吃蛇游戏，包含移动、吃食物、计分、死亡判定。
```
Manager 会自动：创建项目目录 → 分解任务 → 输出 Ready 列表。

### 2. 指派员工（做一次）
在 AI 会话 B 中发送：
```
读取 company-kit/SKILL.md，你是本项目的 Worker。
项目文件夹：snake-game/
```
Worker 会自动：入职 → 查看 Ready 列表 → 认领任务 → 请求审批。

### 3. 推进循环（重复）
在两个会话之间来回发送：

```
会话 A (Manager): "继续检查看板"
→ Manager 处理审批、标记 Done、解锁下游

会话 B (Worker): "继续"
→ Worker 查收审批、执行任务、交付
```

每次“继续”推进一步。重复直到 Manager 报告所有任务 Done。
没有中间评审环节 — Worker 交付后 Manager 直接标记 Done。

### 4. 最终验收（做一次）
Manager 报告项目完成后，你检查交付物，回复：
```
验收通过
```
或提出修改意见。

---

## 简化模式（单会话）

如果你只想用一个 AI 会话完成全部工作：
```
读取 company-kit/SKILL.md，你同时担任 Manager 和 Worker。
项目目标：开发一个浏览器贪吃蛇游戏。
```
然后每次发送"继续"推进一步。模型会自己在两个角色之间切换。

---

## 你需要做的判断

| 时机 | 你要做什么 |
|------|-----------|
| Manager 输出 Ready 列表 | 确认没问题，或要求修改需求 |
| Manager 报告项目完成 | **测试整个项目**，然后验收或拒绝 |
| 有人报告阻塞 | 告诉 Manager 如何解决 |
| 想加新功能 | 告诉 Manager 追加需求 |

---

## 常用指令速查

| 指令 | 发给谁 | 效果 |
|------|--------|------|
| `继续检查看板` | Manager | 检查状态、处理审批、推进流程 |
| `继续` | Worker | 查收指令或审批，执行下一步 |
| `验收通过` | Manager | 最终项目验收 |
| `需要修改：<说明>` | Manager | 追加需求或修改要求 |
| `当前状态？` | Manager | 获取项目进度报告 |
